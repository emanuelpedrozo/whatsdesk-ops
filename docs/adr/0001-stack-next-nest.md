# ADR 0001: Stack Next.js + NestJS

## Status
Aceito

## Contexto
Necessidade de time pequeno entregar rapido com TypeScript ponta a ponta.

## Decisao
- Frontend: Next.js + React + TypeScript.
- Backend: NestJS + TypeScript.

## Consequencias
- Reuso de tipos e menor atrito entre times.
- Curva de aprendizado menor para evoluir de MVP para escala.
